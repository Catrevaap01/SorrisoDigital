1. Introdução e Objectivos

O TeOdonto Angola é uma aplicação multiplataforma de telemedicina dentária desenvolvida para facilitar o acesso a serviços odontológicos em Angola. A aplicação funciona em Web PWA Mobile (Android/iOS) com um único código-fonte.

Objectivos Principais:

  • Triagem Remota: Pacientes podem enviar fotos e descrever sintomas para avaliação à distância.
  • Gestão de Consultas: Agendamento, confirmação e acompanhamento de consultas.
  • Comunicação em Tempo Real: Chat directo entre paciente e dentista.
  • Educação em Saúde: Conteúdos educativos de saúde bucal por categoria.
  • Gestão do Consultório: Dashboard completo para dentistas com métricas e relatórios.
  • Segurança: Sessão única por utilizador, autenticação robusta, dados protegidos.

6. Gestão de Estado Global — src/contexts/

6.1 AuthContext.tsx — O Coração da Aplicação

O AuthContext é o contexto mais complexo do sistema. Gere todo o ciclo de vida da autenticação:

FuncionalidadeDetalhes
Login (signIn)Autentica via supabase.auth.signInWithPassword. Verifica se o servidor está acessível antes de tentar. Gera um session_token único e grava no banco e no storage local. Invalida sessões anteriores com signOut({ scope: 'others' }). Se as credenciais forem inválidas, verifica se o email existe e retorna mensagem contextual.
Sessão ÚnicaAo fazer login, grava last_session_id no perfil do utilizador. Um listener Realtime do Supabase monitora alterações neste campo. Se outro dispositivo fizer login (alterando o token), o dispositivo actual é desconectado instantaneamente com a mensagem "A sua conta foi acedida noutro dispositivo".
Logout (signOut)Limpa estado local imediatamente. Remove last_session_id do banco. Chama signOut({ scope: 'global' }) para invalidar todas as sessões em todos os dispositivos.
Registo (signUp)Cria conta via supabase.auth.signUp. Resolve provincia_id (tentativa exacta, case-insensitive, ou cliente-side). Regista sessão inicial.
Perfil (fetchProfile)Busca dados do utilizador com tratamento de RLS (Row Level Security). Se RLS bloquear, usa um admin client como fallback. Verifica conflito de sessão no carregamento. Se o perfil não existir, cria um registro mínimo automaticamente.
BootstrapNo arranque da app, tenta recuperar a sessão com timeout de 10s. Se o Supabase não estiver acessível, libera a interface sem sessão. Previne travamentos em conexões lentas.

6.2 DentistContext.tsx

Permite ao paciente seleccionar o dentista com quem deseja interagir. O estado selectedDentist é partilhado entre todas as telas do paciente. Inclui lógica de auto-abertura (requestAutoOpenChooseDentist) para forçar a selecção após completar o perfil.

13. Camada de Serviços — src/services/

Cada serviço segue o padrão consistente:

export const buscarDados = async (id: string): Promise<ServiceResult<T>> => {
  try {
    const { data, error } = await withTimeout(
      supabase.from('tabela').select('*').eq('id', id),
      10000 // timeout de 10 segundos
    );
    if (error) throw error;
    return { success: true, data };
  } catch (err) {
    return { success: false, error: err.message };
  }
};

⚠️ O sistema implementa múltiplas camadas de segurança:

22. Como Executar

# Instalar dependências
npm install

# Desenvolvimento (Web)
npx expo start --web

# Desenvolvimento (Mobile)
npx expo start

# Gerar build para PWA
npx expo export

# Servir PWA localmente
cd dist && npx serve -s -p 3000
